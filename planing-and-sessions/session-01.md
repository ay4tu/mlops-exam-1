## Decisions made (feeds ADR-3)

- [x] **Model algorithm**: `HistGradientBoostingClassifier` (scikit-learn built-in). XGBoost excluded — requires `libomp` which is not present on macOS by default. HistGBM is equally fast (histogram-based splits) and achieves equivalent accuracy with no extra system dependency.
- [x] **Features included**: `amt`, `log_amt`, `category_enc`, `gender_enc`, `city_pop`, `lat`, `long`, `merch_lat`, `merch_long`, `distance`, `hour`, `day_of_week`, `age` (13 features). Dropped: name fields, address strings, transaction ID — these are either PII or have no predictive signal at inference time.
- [x] **Why Postgres as the MLflow backend store**: relational schema handles concurrent writes from multiple training runs; Postgres is already a required dependency in the stack (Feast also uses it); SQLite would corrupt under parallel access.
- [x] **Artifact store**: MLflow `--serve-artifacts` proxy mode with Docker volume (`mlruns`). Clients upload via HTTP so the training script needs no direct filesystem access to the container volume. S3 replaces the volume in production (Phase 4).

## Notes

- Port 5000 is occupied by macOS AirPlay Receiver. MLflow exposed on **5001** locally; set `MLFLOW_TRACKING_URI=http://localhost:5001` in `.env`.
- MLflow 3.x requires `--disable-security-middleware` for dev (otherwise binds to 127.0.0.1 only).
- MLflow 3.x artifact proxy: must use `--default-artifact-root mlflow-artifacts:/` so clients upload via HTTP rather than direct filesystem writes.
- Model stages API (`transition_model_version_stage`) is deprecated in MLflow 2.9+ but still works; will migrate to aliases in a later session if needed.
